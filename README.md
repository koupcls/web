# Recipy

**Recipy** — это динамическая веб-платформа, объединяющая в себе аспекты социальной сети, личного блога и кулинарного каталога. Приложение позволяет пользователям искать подходящие рецепты для повседневной готовки или вдохновения, а также делиться собственным кулинарным опытом, создавая пошаговые инструкции.

## Целевая аудитория

* Люди всех возрастов, заинтересованные в кулинарии в любой степени.
* Пользователи, ищущие быстрый рецепт на ужин или желающие вспомнить точные пропорции ингредиентов для базовых блюд.
* Творческие кулинары и фуд-блогеры, которые хотят удобно оформлять свои рецепты и делиться ими с аудиторией.

## Ключевой функционал

* **Интеллектуальная лента и каталог:** Просмотр рецептов с удобным поиском по ингредиентам, кухне мира, тегам, названию и авторам.
* **Личный профиль пользователя:** Возможность сохранять понравившиеся рецепты в закладки, отслеживать свои оценки и комментарии.
* **Продвинутый конструктор постов:** * Добавление названия, описания и медиафайлов.
* Формирование пошаговой инструкции по приготовлению.
* Удобное добавление списка ингредиентов (с выбором типа пропорций: граммы, столовые ложки, штуки и т.д.).
* Присвоение тегов (например, "сладкое", "острое", "итальянская кухня").


* **Социальное взаимодействие:** Возможность оценивать рецепты, оставлять комментарии и подписываться на авторов.

## Структура приложения

| Название раздела | Описание | Тип доступа | Маршрут (Frontend) |
| --- | --- | --- | --- |
| **Главная** | Шапка, быстрый поиск по категориям, лента популярных или новых рецептов. | Публичный | `/` |
| **Аутентификация** | Страницы регистрации, входа в аккаунт и восстановления пароля. Считается за 1 логический раздел. | Публичный | `/auth/*` |
| **Каталог (Поиск)** | Страница с результатами поиска рецептов по заданным критериям, фильтрам и тегам. | Публичный | `/search` |
| **Подборки** | Динамическая страница с рецептами определенной категории (например, "Рецепты дня" или "Десерты"). | Публичный | `/category/:slug` |
| **Просмотр поста** | Детальная страница конкретного рецепта с пошаговой инструкцией и комментариями. | Публичный | `/post/:id` |
| **Профиль пользователя** | Публичная часть (аватар, описание, опубликованные рецепты) и приватная (закладки, история оценок). | Смешанный | `/profile/:id` |

## Технологический стек

### Frontend

* **Фреймворк: Vue 3 (Composition API)**. Обеспечивает высокую производительность и реактивность интерфейса. Composition API позволяет удобно переиспользовать логику (например, логику добавления ингредиентов в конструкторе), что критично для сложных форм.
* **State-менеджмент: Pinia**. Современный стандарт для Vue, обеспечивающий строгую типизацию и удобное управление глобальным состоянием (хранение данных авторизованного пользователя, его закладок и корзины ингредиентов).
* **Роутинг: Vue Router**. Необходим для создания Single Page Application (SPA), обеспечивает плавный переход между разделами без перезагрузки страницы и позволяет легко настроить защиту приватных маршрутов (navigation guards).
* **HTTP-клиент: Axios**. Библиотека для работы с API. В отличие от стандартного fetch, позволяет удобно настраивать базовый URL сервера и централизованно обрабатывать ошибки ответов.

### Backend

* **Среда выполнения и фреймворк: Node.js + Express**. Асинхронная неблокирующая архитектура Node.js отлично подходит для I/O операций, таких как запросы к БД рецептов. Express предоставляет минималистичный и гибкий инструментарий для быстрого построения REST API эндпоинтов.
* **Инструментарий: dotenv, nodemon, cors.** dotenv обеспечивает безопасное хранение ключей доступа к БД; nodemon ускоряет разработку за счет авто-перезагрузки сервера; cors разрешает взаимодействие между фронтендом и бэкендом.
* **База данных и Backend-as-a-Service: Supabase (PostgreSQL)**. Использование строгой реляционной базы данных (PostgreSQL) идеально подходит для связи сущностей (Пользователь -> Рецепт -> Комментарии -> Ингредиенты). Supabase берет на себя рутину по настройке безопасной аутентификации пользователей и предоставляет удобное хранилище (Storage) для загрузки изображений рецептов.

## Визуальное проектирование интерфейса

### Страница: Главная (/)

**Тип доступа:** публичный

**Основные блоки:**
| № | Блок | Описание | Источник данных |
|---|------|----------|----------------|
| 1 | Шапка | Логотип, кнопка меню (поиск, подборки, уведомления, сохранения), профиль/кнопка "Войти" | `GET /api/user/me` (если авторизован) |
| 2 | Герой-секция | Заголовок, краткое описание функционала, CTA-кнопка "Начать" / "Зарегистрироваться" | Статичный контент |
| 3 | Предложение регистрации | Блок с преимуществами аккаунта (сохранение, оценки, персонализация) | Статичный контент |
| 4 | Рекомендуемые подборки | Карусель из 4-6 тематических подборок с обложками | `GET /api/categories` |
| 5 | Поиск (плавающий) | Поле ввода с автодополнением, появляется при скролле | `GET /api/search` |
| 6 | Лента рекомендаций | Бесконечная лента карточек рецептов | `GET /api/recipes/feed` |
| 7 | Футер | Ссылки на разделы, контакты, соцсети | Статичный контент |

**Динамический контент и состояния:**
```
🔹 Лента рекомендаций:
   • Загрузка: скелетоны карточек (6 шт.)
   • Пусто: "Пока нет рекомендаций" + кнопка "Исследовать подборки"
   • Ошибка: "Не удалось загрузить" + кнопка "Повторить"
   • Подгрузка: спиннер внизу при достижении конца списка

🔹 Рекомендуемые подборки:
   • Загрузка: скелетоны карусели
   • Ошибка: скрыть блок или показать заглушку "Популярные категории"

🔹 Поиск (автодополнение):
   • Ввод: дебаунс 300мс → запрос /api/search/suggest
   • Результаты: выпадающий список (рецепты + категории)
   • Пусто: "Ничего не найдено по запросу '...'"
```

**Поведение для разных пользователей:**
- **Гость:** видит секции 1→2→3→4→6 (плавный скролл), поиск активируется при прокрутке
- **Авторизованный:** начинается с секции 4 (подборки), затем сразу лента 6, поиск зафиксирован сверху


### Страница: Аутентификация (/auth/*)

**Тип доступа:** публичный

**Подстраницы:** `/auth/login`, `/auth/register`, `/auth/recover`, `/auth/logout`

#### Регистрация (/auth/register)
**Основные блоки:**
| № | Блок | Описание | Источник данных |
|---|------|----------|----------------|
| 1 | Шаг 1: Контакты | Поля: email, пароль (с проверкой сложности), подтверждение пароля | Валидация на клиенте + `POST /api/auth/check-email` |
| 2 | Шаг 2: Профиль | Поля: имя пользователя, уникальный тег (@username), аватар (опционально) | `GET /api/auth/check-tag` (проверка уникальности) |
| 3 | Шаг 3: Предпочтения | Опрос: выбор отношения к ингредиентам/кухням (Обожаю/Нравится/Нейтрален/Не люблю/Не ем) | `GET /api/preferences/categories` (список категорий) |
| 4 | Завершение | Кнопка "Создать аккаунт", ссылка "Уже есть аккаунт? Войти" | `POST /api/auth/register` (отправка всех данных) |

**Состояния:**
```
• Валидация полей: подсветка ошибок в реальном времени
• Отправка: спиннер на кнопке, блокировка формы
• Успех: редирект на / с тостом "Добро пожаловать!"
• Ошибка: отображение сообщения под соответствующим полем
```

#### Вход (/auth/login)
**Основные блоки:**
| № | Блок | Описание | Источник данных |
|---|------|----------|----------------|
| 1 | Форма входа | Поля: email/тег, пароль; чекбокс "Запомнить меня" | `POST /api/auth/login` |
| 2 | Дополнительные ссылки | "Забыли пароль?", "Создать аккаунт" | Статичные ссылки |

**Состояния:**
```
• Неправильные данные: ошибка "Неверный email или пароль"
• Слишком много попыток: капча / временная блокировка
• Успех: сохранение токена, редирект на предыдущую страницу или /
```

#### Восстановление пароля (/auth/recover)
**Основные блоки:**
| № | Блок | Описание | Источник данных |
|---|------|----------|----------------|
| 1 | Ввод почты | Поле email, кнопка "Отправить код" | `POST /api/auth/recover/request` |
| 2 | Ввод кода | 6-значный код из письма, таймер повторной отправки (60 сек) | `POST /api/auth/recover/verify` |
| 3 | Новый пароль | Поля: новый пароль, подтверждение | `POST /api/auth/recover/reset` |

#### Выход (/auth/logout)
**Основные блоки:**
| № | Блок | Описание | Источник данных |
|---|------|----------|----------------|
| 1 | Подтверждение | Текст "Вы уверены, что хотите выйти?", кнопки "Выйти" / "Отмена" | `POST /api/auth/logout` (инвалидация токена) |



### Страница: Профиль (/profile/:id)

**Тип доступа:** смешанный (публичный просмотр, приватное редактирование)

**Основные блоки (общие):**
| № | Блок | Описание | Источник данных |
|---|------|----------|----------------|
| 1 | Хедер профиля | Аватар, имя, тег, дата регистрации, статистика (посты/оценки/подписчики) | `GET /api/users/:id` |
| 2 | Боковое меню | Навигация по вкладкам: Предпочтения, Посты, Сохранения, Оценки, Уведомления, Настройки | Статичное меню |
| 3 | Область контента | Динамически меняется в зависимости от выбранной вкладки | Зависит от вкладки |
| 4 | Футер | Ссылки, контакты | Статичный контент |

#### Вкладка: Вкусовые предпочтения (.../preferences)
**Тип доступа:** публичный

| Блок | Описание | Источник данных |
|------|----------|----------------|
| Категории предпочтений | Аккордеон: Ингредиенты / Кухни / Специи / Блюда | `GET /api/preferences/categories` |
| Элемент категории | Название + 5 кнопок-эмодзи (Обожаю / Нравится / Нейтрален / Не люблю / Не ем) | `GET /api/users/:id/preferences` |
| Статистика | Визуализация: "80% ваших предпочтений совпадают с этим пользователем" | Вычисляется на клиенте |

**Динамика:**
- Если текущий пользователь == владелец профиля: кнопки интерактивны → `PATCH /api/users/me/preferences`
- Если гость: только просмотр, кнопки неактивны

#### Вкладка: Посты пользователя (.../posts)
**Тип доступа:** публичный

| Блок | Описание | Источник данных |
|------|----------|----------------|
| Сетка постов | Карточки рецептов (изображение, заголовок, рейтинг, дата) | `GET /api/users/:id/posts` |
| Фильтры (опционально) | Сортировка: новые/популярные; фильтр по тегу | Параметры запроса |
| Пагинация | Кнопка "Загрузить ещё" или бесконечный скролл | `?offset=..` |

**Права доступа:**
```
• Автор поста: видит кнопки "Редактировать" и "Удалить" на своих карточках
• Гость: только просмотр, клик → переход на /post/:id
```

#### Вкладка: Сохранения (.../saves)
**Тип доступа:** смешанный (пользователь настраивает публичность подвкладок)

| Подвкладка | Описание | Источник данных | Публичный доступ |
|------------|----------|----------------|-----------------|
| Хочу приготовить | Рецепты для будущего приготовления | `GET /api/users/:id/saves?folder=wants` | Настраивается |
| Понравилось | Оценённые положительно рецепты | `GET /api/users/:id/saves?folder=liked` | Настраивается |
| Не понравилось | Рецепты с низкой оценкой | `GET /api/users/:id/saves?folder=disliked` | Приватно |
| Заметки | Приватные черновики собственных рецептов | `GET /api/users/:id/notes` | Приватно |

**Особенность подвкладки "Заметки":**
- Кнопка "+ Создать заметку" → переход в упрощённый конструктор
- Заметки отображаются как мини-карточки с превью текста
- Возможность одним кликом "Опубликовать как рецепт" → переход в полный конструктор

#### Вкладка: Оценки (.../ratings)
**Тип доступа:** публичный

| Блок | Описание | Источник данных |
|------|----------|----------------|
| Список оценок | Карточки рецептов + выбранная оценка (1-5) + дата оценки | `GET /api/users/:id/ratings` |
| Фильтры | По диапазону дат, по оценке (только 5, только 1-2 и т.д.) | Параметры запроса |

#### Вкладка: Уведомления (.../notifications)
**Тип доступа:** приватный (только владелец профиля)

| Блок | Описание | Источник данных |
|------|----------|----------------|
| Список уведомлений | Карточки: тип (иконка), текст, время, статус прочтения | `GET /api/users/me/notifications` |
| Фильтры | Все / Непрочитанные / По типу (комментарии, оценки, система) | Параметры запроса |
| Массовые действия | "Отметить все как прочитанные", "Очистить старые" | `PATCH /api/users/me/notifications/read-all` |

**Типы уведомлений:**
```
• Пост: опубликован / изменён / удалён
• Взаимодействие: ваш пост оценили / прокомментировали / сохранили
• Подписки: на вас подписались / отписались / новый пост у автора
• Системные: обновление правил, новые функции
```

#### Вкладка: Настройки (.../settings)
**Тип доступа:** приватный

| Блок | Описание | Источник данных |
|------|----------|----------------|
| Профиль | Редактирование: имя, тег, аватар, био | `PATCH /api/users/me` |
| Безопасность | Смена пароля, привязка/отвязка почты, 2FA (опционально) | `POST /api/users/me/password` |
| Приватность | Настройки видимости вкладок "Сохранения", кто может комментировать | `PATCH /api/users/me/privacy` |
| Уведомления | Вкл/выкл пуш-уведомлений по типам событий | `PATCH /api/users/me/notification-settings` |
| Опасная зона | "Деактивировать аккаунт", "Удалить аккаунт" (с подтверждением) | `DELETE /api/users/me` |

### Страница: Поиск (/search)

**Тип доступа:** публичный

**Основные блоки:**
| № | Блок | Описание | Источник данных |
|---|------|----------|----------------|
| 1 | Поле поиска | Ввод запроса с автодополнением | `GET /api/search` |
| 2 | Фильтры | Категория, время приготовления, сложность, рейтинг, теги | `GET /api/filters/options` |
| 3 | Сортировка | По релевантности / новизне / рейтингу / просмотрам | Параметр `?sort=...` |
| 4 | Результаты | Сетка карточек рецептов (изображение, заголовок, автор, рейтинг) | `GET /api/search?...` |

**Динамика и состояния:**
```
🔹 Автодополнение:
   • Ввод "борщ" → запрос /api/search/suggest?q=борщ
   • Отображение: выпадающий список с иконками типов

🔹 Результаты поиска:
   • Загрузка: скелетоны сетки
   • Пусто: "Ничего не найдено по запросу '...'" + рекомендации сбросить фильтры
   • Ошибка: "Проблема с подключением" + кнопка повтора

🔹 Применение фильтров:
   • Изменение любого фильтра → автоматический перезапрос результатов
```

### Страница: Просмотр поста (/post/:id)

**Тип доступа:** публичный

**Основные блоки:**
| № | Блок | Описание | Источник данных |
|---|------|----------|----------------|
| 1 | Заголовок | Название рецепта, автор (ссылка на профиль), дата публикации, рейтинг | `GET /api/recipes/:id` |
| 2 | Галерея | Основное изображение + миниатюры шагов (свайп/зум) | Вложенный массив `images[]` в ответе API |
| 3 | Мета-информация | Время приготовления, сложность, порции, теги | Поля рецепта |
| 4 | Ингредиенты | Список: название, количество, единица измерения; чекбокс "Есть в наличии" | Поле `ingredients[]` |
| 5 | Шаги приготовления | Нумерованный список с текстом и опциональными изображениями | Поле `steps[]` |
| 6 | Взаимодействие | Кнопки: Оценка (1-5), Сохранить, Поделиться, Комментарии | `POST /api/recipes/:id/rate`, `POST /api/recipes/:id/save` |
| 7 | Комментарии | Список комментариев, форма добавления (для авторизованных) | `GET /api/recipes/:id/comments`, `POST /api/recipes/:id/comments` |

**Динамика и состояния:**
```
🔹 Оценка рецепта:
   • Неавторизованный: клик → модалка "Войдите, чтобы оценить"
   • Авторизованный: выбор звезды → анимация → запрос POST /api/recipes/:id/rate
   • Повторная оценка: обновление выбранного значения

🔹 Сохранение:
   • Клик по кнопке сохранить → выбор папки (Хочу приготовить / Понравилось / Заметки) → сохранение
   • Визуальная обратная связь: иконка меняет цвет/заполнение

🔹 Комментарии:
   • Подгрузка: пагинация по 10 штук или бесконечный скролл
   • Отправка: валидация (мин. 3 символа), спиннер, добавление в начало списка без перезагрузки

🔹 Состояния страницы:
   • Загрузка: скелетон заголовка + галереи + контента
   • Не найдено: 404-страница с предложением "Посмотреть другие рецепты"
   • Ошибка сети: "Не удалось загрузить рецепт" + кнопка повтора
```

### Страница: Конструктор постов (/post/create)

**Тип доступа:** приватный (только для авторизованных)

**Основные блоки:**
| № | Блок | Описание | Источник данных |
|---|------|----------|----------------|
| 1 | Заголовок | Поле ввода названия рецепта (обязательно, 3-100 символов) | Клиентская валидация |
| 2 | Изображения | Drag&Drop зона + выбор файлов; превью с возможностью удаления | Загрузка на `POST /api/upload/image` → получение URL |
| 3 | Ингредиенты | Динамический список: [название] [количество] [единица]; кнопки "+ Добавить" / "Удалить" | Клиентское управление массивом |
| 4 | Шаги приготовления | Нумерованные блоки: текст + опциональное изображение; сортировка перетаскиванием | Клиентское управление массивом |
| 5 | Мета-данные | Выбор категории, тегов (выбор из популярных + ввод своих), время, сложность | `GET /api/categories`, `GET /api/tags/popular` |
| 6 | Настройки публикации | Чекбоксы: "Опубликовать сразу" / "Сохранить как черновик", "Разрешить комментарии" | Параметры отправки |
| 7 | Предпросмотр | Кнопка "Предпросмотр" → модалка с тем, как будет выглядеть пост | `POST /api/recipes/preview` (опционально) |
| 8 | Действия | Кнопки: "Опубликовать", "Сохранить черновик", "Отмена" | `POST /api/recipes` |

**Состояния формы:**
```
• Валидация в реальном времени: подсветка пустых обязательных полей при попытке отправки
• Загрузка изображений: прогресс-бар для каждого файла
• Отправка: блокировка кнопок, спиннер, текст "Публикуем..."
• Успех: редирект на /post/:id с тостом "Рецепт опубликован!"
• Ошибка: скролл к первому проблемному полю, отображение сообщения
```

### Страница: Подборки (/category/:slug)

**Тип доступа:** публичный

**Основные блоки:**
| № | Блок | Описание | Источник данных |
|---|------|----------|----------------|
| 1 | Хедер подборки | Название, описание, обложка, статистика (количество рецептов) | `GET /api/categories/:slug` |
| 2 | Фильтры внутри категории | По тегам, рейтингу, времени приготовления | Параметры запроса |
| 3 | Сетка рецептов | Карточки рецептов, отсортированные по популярности/новизне | `GET /api/categories/:slug` |

**Динамика:**
```
• Применение фильтра → перезапрос списка без перезагрузки страницы
• Состояние "Пусто": "В этой подборке пока нет рецептов" + предложение посмотреть другие категории
• Загрузка: скелетоны сетки
```

# Схема данных и API

## Модели данных

###  User (Пользователь)
| Поле | Тип | Обязательное | Описание |
|------|-----|--------------|----------|
| `id` | UUID | ✅ | Первичный ключ |
| `email` | STRING | ✅ | Уникальный, используется для входа и восстановления |
| `username` | STRING | ✅ | Отображаемое имя |
| `tag` | STRING | ✅ | Уникальный короткий идентификатор, формат @tag |
| `avatar_url` | STRING | ❌ | Ссылка на изображение профиля |
| `preferences` | JSONB | ❌ | Структура вкусовых предпочтений |
| `created_at` | TIMESTAMP | ✅ | Дата и время регистрации |
| `updated_at` | TIMESTAMP | ✅ | Дата последнего обновления профиля |

### Recipe (Рецепт/Пост)
| Поле | Тип | Обязательное | Описание |
|------|-----|--------------|----------|
| `id` | UUID | ✅ | Первичный ключ |
| `title` | STRING | ✅ | Название рецепта (3-100 символов) |
| `description` | TEXT | ❌ | Краткое описание (до 300 символов) для превью |
| `ingredients` | JSONB | ✅ | Массив: `[{ "name": "Мука", "amount": 200, "unit": "г" }]` |
| `steps` | JSONB | ✅ | Массив шагов: `[{ "text": "Смешать...", "image_url": "https://..." }]` |
| `images` | JSONB | ❌ | Массив URL изображений: `["https://...", "..."]` |
| `author_id` | UUID | ✅ | Внешний ключ → User.id |
| `category_id` | UUID | ❌ | Внешний ключ → Category.id (опционально) |
| `tags` | STRING[] | ❌ | Массив тегов: `["завтрак", "быстро", "вегетарианское"]` |
| `prep_time_min` | INTEGER | ❌ | Время приготовления в минутах |
| `rating_avg` | FLOAT | ✅ (по умолчанию 0) | Средний рейтинг (кэшируется, обновляется триггером) |
| `rating_count` | INTEGER | ✅ (по умолчанию 0) | Количество оценок |
| `views_count` | INTEGER | ✅ (по умолчанию 0) | Количество просмотров |
| `saves_count` | INTEGER | ✅ (по умолчанию 0) | Количество сохранений |
| `created_at` | TIMESTAMP | ✅ | Дата создания |
| `updated_at` | TIMESTAMP | ✅ | Дата последнего изменения |
