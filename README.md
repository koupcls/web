# Recipy

**Recipy** — это динамическая веб-платформа, объединяющая в себе аспекты социальной сети, личного блога и кулинарного каталога. Приложение позволяет пользователям искать подходящие рецепты для повседневной готовки или вдохновения, а также делиться собственным кулинарным опытом, создавая пошаговые инструкции.

## Целевая аудитория

* Люди всех возрастов, заинтересованные в кулинарии в любой степени.
* Пользователи, ищущие быстрый рецепт на ужин или желающие вспомнить точные пропорции ингредиентов для базовых блюд.
* Творческие кулинары и фуд-блогеры, которые хотят удобно оформлять свои рецепты и делиться ими с аудиторией.

## Ключевой функционал

* **Интеллектуальная лента и каталог:** Просмотр рецептов с удобным поиском по ингредиентам, кухне мира, тегам, названию и авторам.
* **Личный профиль пользователя:** Возможность сохранять понравившиеся рецепты в закладки, отслеживать свои оценки и комментарии.
* **Продвинутый конструктор постов:** * Добавление названия, описания и медиафайлов.
* Формирование пошаговой инструкции по приготовлению.
* Удобное добавление списка ингредиентов (с выбором типа пропорций: граммы, столовые ложки, штуки и т.д.).
* Присвоение тегов (например, "сладкое", "острое", "итальянская кухня").


* **Социальное взаимодействие:** Возможность оценивать рецепты, оставлять комментарии и подписываться на авторов.

## Структура приложения

| Название раздела | Описание | Тип доступа | Маршрут (Frontend) |
| --- | --- | --- | --- |
| **Главная** | Шапка, быстрый поиск по категориям, лента популярных или новых рецептов. | Публичный | `/` |
| **Аутентификация** | Страницы регистрации, входа в аккаунт и восстановления пароля. Считается за 1 логический раздел. | Публичный | `/auth/*` |
| **Каталог (Поиск)** | Страница с результатами поиска рецептов по заданным критериям, фильтрам и тегам. | Публичный | `/search` |
| **Подборки** | Динамическая страница с рецептами определенной категории (например, "Рецепты дня" или "Десерты"). | Публичный | `/category/:slug` |
| **Просмотр поста** | Детальная страница конкретного рецепта с пошаговой инструкцией и комментариями. | Публичный | `/post/:id` |
| **Профиль пользователя** | Публичная часть (аватар, описание, опубликованные рецепты) и приватная (закладки, история оценок). | Смешанный | `/profile/:id` |
| **Настройки профиля** | Редактирование личных данных, аватара, вкусовых предпочтений. | Приватный | `/settings` |
| **Конструктор постов** | Интерфейс для создания нового рецепта или редактирования уже опубликованного. | Приватный | `/post/create` |

## Технологический стек

### Frontend

* **Фреймворк:** **Vue 3 (Composition API)**. Обеспечивает высокую производительность и реактивность интерфейса. Composition API позволяет удобно переиспользовать логику (например, логику добавления ингредиентов в конструкторе), что критично для сложных форм.
* **State-менеджмент:** **Pinia**. Современный стандарт для Vue, обеспечивающий строгую типизацию и удобное управление глобальным состоянием (хранение данных авторизованного пользователя, его закладок и корзины ингредиентов).
* **Роутинг:** **Vue Router**. Необходим для создания Single Page Application (SPA), обеспечивает плавный переход между разделами без перезагрузки страницы и позволяет легко настроить защиту приватных маршрутов (navigation guards).

### Backend

* **Среда выполнения и фреймворк: Node.js + Express**. Асинхронная неблокирующая архитектура Node.js отлично подходит для I/O операций, таких как запросы к БД рецептов. Express предоставляет минималистичный и гибкий инструментарий для быстрого построения REST API эндпоинтов.
* **База данных и Backend-as-a-Service: Supabase (PostgreSQL)**. Использование строгой реляционной базы данных (PostgreSQL) идеально подходит для связи сущностей (Пользователь -> Рецепт -> Комментарии -> Ингредиенты). Supabase берет на себя рутину по настройке безопасной аутентификации пользователей и предоставляет удобное хранилище (Storage) для загрузки изображений рецептов.
